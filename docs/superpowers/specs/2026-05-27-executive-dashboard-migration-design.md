# 高管工作台迁移设计

## 目标

把 `gaoguan/` 目录里的高管驾驶舱迁移到当前 IT 项目管理系统的高管工作台中。迁移后的页面必须保留现有系统侧边栏、顶部栏、角色切换和主题切换，不作为独立全屏 demo 或 iframe 存在。

## 设计原则

- 高管工作台替换当前高管进入 `项目驾驶舱` 时看到的内容。
- 管理员可以复用同一套高管驾驶舱视图，用于全量查看。
- PC 端占满现有主内容区域，适配侧边栏展开和收起，不再使用 `gaoguan` 独立页面的居中窄容器。
- 默认主题为 `dark`，并提供 `light` 版本，跟随现有系统的主题切换。
- 数据优先从当前系统的 `projects`、`demands`、`tasks`、`flows` 派生，避免形成一套无法互相印证的孤立 mock。

## 页面内容

高管工作台保留 `gaoguan` 的决策型信息结构，但转成当前系统口径：

- 顶部 KPI：需求总量、进行中项目、预算使用、资源投入、高风险项目、待高管关注事项。
- 需求承接总览：按需求状态统计，并展示业务部门、IT 部、管理层等当前组织口径的热力分布。
- 资源投入情况：展示 IT 项目治理、IT 内部开发、业务部门验收投入、外部供应商投入和预算执行。
- 项目交付路线图：按项目展示月度甘特条，点击项目可进入当前项目详情。
- 风险解读：从项目风险、预算偏差、延期任务和资源过载中生成高管可读的风险卡片。

## 交互

- 部门/范围筛选保留为页面内轻量筛选，不越权。
- 月份切换用于甘特视图展示窗口。
- 项目条、风险卡片和项目排行中的项目名称可打开项目详情。
- 主题切换仍使用系统已有 `light / dark` 控件；高管默认进入 dark。

## 非目标

- 不迁移 `gaoguan/node_modules`、`dist` 或独立 Vite 工程结构。
- 不引入图表库。
- 不新增真实后端、BI 查询、权限鉴权或企业微信接口。
- 不让高管页面脱离当前系统布局。

## 验收

- `npm run build` 通过。
- `git diff --check` 通过。
- 高管角色默认打开暗色高管驾驶舱。
- 切换到 light 后，高管驾驶舱有对应浅色样式。
- 页面在现有侧边栏展开和收起时占满主内容区域。
- 点击高管驾驶舱中的项目可以进入现有项目详情。
