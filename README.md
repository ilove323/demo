# IT Project Management Prototype

医药企业 IT 项目管理系统静态原型，基于 Vite + React + TypeScript 实现。原型以 `docs/IT项目管理系统需求文档V1.3.docx` 为最新需求来源，采用单套 H5 响应式网页，用于演示 IT 项目从需求提交、产品评估、项目申请、资源排期、任务实施、验收到上线归档的协作流程。

## 功能范围

- H5 后台：项目驾驶舱、需求管理、产品工作台、项目管理、任务与工时、资源与预算、绩效与报表、个人消息通知、系统设置。
- 响应式适配：桌面端使用左侧导航，手机宽度和企业微信内嵌浏览器下同一套页面转为顶部横向导航和单列内容。
- 多角色演示：管理员、高管、项目经理、IT负责人、产品经理、运营、运营中心负责人、开发、研发负责人。
- 数据联动：需求、项目、AI 立项评分、流程、任务、资源排期、供应商预算、通知和报表使用同一批 mock 数据互相印证。
- AI 立项评分：项目列表、项目申请池和项目详情均显性展示业务价值、紧急程度、可行性、加权总分、推荐结论和理由。
- 详情页：需求和项目列表保持清爽摘要，完整业务内容进入应用内详情页，不再用侧边栏承载需求/项目详情。
- 甘特视图：项目管理支持团队项目甘特，资源与预算支持 IT 内部资源甘特，窄屏下在自身容器内横向滚动。

## 环境要求

建议使用：

- Node.js 20+
- npm 10+

查看本机版本：

```bash
node -v
npm -v
```

## 安装依赖

首次拉取项目后，在仓库根目录执行：

```bash
npm install
```

## 本地开发

启动开发服务器：

```bash
npm run dev
```

Vite 会输出本地访问地址，通常是：

```text
http://localhost:5173/
```

如果 5173 被占用，Vite 会自动使用下一个可用端口，例如 `5174`。

## 编译构建

执行 TypeScript 检查并生成生产构建：

```bash
npm run build
```

构建产物会输出到：

```text
dist/
```

## 预览构建产物

构建后可以用 Vite 预览生产产物：

```bash
npm run preview
```

然后打开终端输出的预览地址。

## 使用说明

打开页面后：

- 左侧底部 `切换身份` 可切换不同角色视角，手机宽度下会出现在顶部导航区域。
- 左侧导航会按当前角色权限过滤，不同角色看到的模块不同。
- 顶部可切换 `light / dark` 商务主题。
- 页面使用单套 H5 响应式布局，桌面、平板、手机宽度和企业微信内嵌浏览器访问同一个业务界面。

## 原型边界

当前项目是静态前端原型：

- 不接后端。
- 不接数据库。
- 不做真实登录和真实鉴权。
- 不调用真实企业微信、MCP、AI 评分或机器人接口。
- 页面操作只在前端内存中临时生效，刷新后恢复初始 mock 数据。

## 开发约定

- 新增 mock 数据时，必须保证需求、项目、流程、任务、资源排期和通知之间能互相印证。
- 不要在单个页面新增孤立数据；例如项目里的关联任务必须能在任务列表中找到，资源甘特里的任务必须能对应真实 `taskId/projectId`。
- 新增重要业务能力时，必须保证同一套 H5 页面在桌面和手机宽度下都可访问、无重叠。
- 修改稳定业务口径时，同步更新 `docs/` 下的总览和演进故事文档。
